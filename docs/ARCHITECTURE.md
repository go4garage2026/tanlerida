## рҹҺҜ Overview

TANLERIDA is built as a **modern full-stack web application** using the following architectural principles:

- **Server-Side Rendering (SSR)** for SEO and performance
- **API-first design** for flexibility and future mobile apps
- **Microservices-ready** AI pipeline for independent scaling
- **Event-driven architecture** for async operations
- **Cloud-native** deployment on Vercel

#### Component Architecture
- **Server Components** (default): Product lists, static content
- **Client Components** ('use client'): Interactive elements, forms
- **Compound Components**: Complex UI like cart drawer
- **Custom Hooks**: Reusable logic (useCart, useAuth)

### 2. Backend (API Routes)

#### API Design Principles
- **RESTful** with predictable URLs
- **Type-safe** with Zod validation
- **Error handling** with consistent response format
- **Rate limiting** on sensitive endpoints

```typescript
// Example: Standard API Response Format
interface ApiResponse<T> {
  success: boolean;
  data?: T;
  error?: {
    code: string;
    message: string;
    details?: unknown;
  };
  meta?: {
    page: number;
    limit: number;
    total: number;
  };
}
```

### 3. Database (PostgreSQL + Prisma)

#### Schema Design
```prisma
// Core entities with relationships
model User {
  id          String   @id @default(cuid())
  email       String   @unique
  orders      Order[]
  cart        Cart?
  TanLeridaSessions TanLeridaSession[]
  // ...
}

model Product {
  id          String   @id @default(cuid())
  slug        String   @unique
  name        String
  price       Int      // Paise (вӮ№1 = 100 paise)
  inventory   Int
  category    Category @relation(fields: [categoryId], references: [id])
  // ...
}

model Order {
  id          String   @id @default(cuid())
  user        User     @relation(fields: [userId], references: [id])
  items       OrderItem[]
  payment     Payment?
  status      OrderStatus
  // ...
}
```

#### Indexing Strategy
- Primary keys: `id` fields (CUID)
- Unique constraints: `email`, `slug`, `sku`
- Foreign keys: Automatic indexes on relations
- Search indexes: `name` (GIN for full-text)

### Security Measures

| Layer | Implementation |
|-------|----------------|
| **Authentication** | NextAuth.js v5 with JWT strategy |
| **Password Hashing** | bcryptjs with salt rounds 12 |
| **Session Management** | HTTP-only cookies, secure flag |
| **CSRF Protection** | Built-in NextAuth.js CSRF tokens |
| **Rate Limiting** | Custom middleware on API routes |
| **Input Validation** | Zod schemas for all inputs |
| **SQL Injection** | Prisma ORM (parameterized queries) |
| **XSS Protection** | React escaping, CSP headers |

### Environment Security
- No secrets in codebase
- `.env.local` in `.gitignore`
- Production secrets in Vercel Environment Variables
- Separate dev/staging/prod configurations
